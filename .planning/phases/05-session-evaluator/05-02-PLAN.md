**Skill body — exact behavior to implement:**

The skill instructs Claude to perform these steps at session start:

**Step 1 — Check for pending drafts**
Use the Bash tool to list JSON files in `.engram/pending_memories/` relative to the current working directory:

  ls .engram/pending_memories/*.json 2>/dev/null

If no files found (command fails or returns nothing): print a single line "No pending Engram memory drafts." and stop.

If files exist: proceed to Step 2.

**Step 2 — Present each draft**
For each pending file, read it with the Bash tool:

  cat .engram/pending_memories/{filename}

Parse the JSON and present the draft in this format:

  Pending Engram Memory Draft ({N} of {total}):

    Key:        {draft_key}
    Title:      {draft_title}
    Tags:       {draft_tags joined with ", "}
    Confidence: {confidence} ({reasoning})
    Evaluated:  {evaluated_at}

    Content preview:
    {draft_content — first 400 characters, ellipsis if truncated}

    [Dedup warning: similar memory "{existing_key}" exists (score={score})]
    (only show the dedup warning line if dedup_warning is not null)

  Store this memory? (approve / skip / edit)

**Step 3 — Handle response**

- "approve" or similar: call mcp__engram__store_memory with key=draft_key, content=draft_content, title=draft_title, tags=comma-separated draft_tags string, force=true. After successful store: delete the pending file using Bash (rm .engram/pending_memories/{filename}). Report: "Stored: {draft_title}".

- "skip" or similar: leave the file in place. Report: "Skipped — draft will appear again next session."

- "edit" or corrections: update the relevant fields in the draft presentation, re-show the draft. Ask again. Do NOT store until the user explicitly approves.

- "delete" or explicit discard: delete the pending file without storing. Report: "Discarded: {draft_key}."

**Step 4 — Continue to next draft**
After handling one draft, present the next one. Repeat until all drafts have been handled or the user says "stop" / "done".

**Conventions — always enforced:**
- Never call store_memory without explicit approval
- Always pass force=true to store_memory (dedup was already checked by evaluator)
- Tags must be passed as comma-separated string (e.g. "engram,tooling,pattern")
- If store_memory returns an error, report it and ask whether to retry or skip
