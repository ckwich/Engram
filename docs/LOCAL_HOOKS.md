# Local Git Hooks

Engram's codebase mapping hook is an optional post-commit hook. It is installed
only after explicit operator intent through `install_codebase_mapping_hook` or:

```bash
python engram_index.py --project /path/to/project --install-hook
```

The hook runs `engram_index.py --mode evolve` after commits and writes its log
under the target repo's `.engram/` directory. It should never block a commit.

## Platform Hygiene

Git hooks live under `.git/hooks/`, so they are local checkout state rather than
tracked repository files. After a cross-machine migration, a stale Windows hook
can remain in a macOS checkout with `C:\...` paths or a Git Bash shebang. In
that case, disable the hook rather than chmod it:

```bash
mv .git/hooks/post-commit .git/hooks/post-commit.disabled-windows
```

Reinstall the hook only when this checkout should run codebase mapping after
each commit. A valid installed hook should be generated by Engram on the current
machine, use this checkout's Python and `engram_index.py` paths, and be
executable.
